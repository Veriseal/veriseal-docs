# Parte II – La Arquitectura de VeriSeal

La Parte II describe la arquitectura técnica que transforma a VeriSeal de una tesis conceptual de integridad en una infraestructura desplegable.

La arquitectura está diseñada para satisfacer tres restricciones simultáneas:

1. **Integridad criptográfica determinista**
2. **Robustez operativa de nivel institucional**
3. **Verificabilidad pública independiente**

VeriSeal no está construido como una blockchain.
Está construido como una capa de integridad estructurada.

Esta sección cubre:

- Modelo de canonización y hash  
- Agregación de Merkle y vinculación de múltiples artefactos  
- Estructura de libro mayor de solo anexión  
- Arquitectura de verificación pública  
- Modelo de soberanía y despliegue  
- Posicionamiento de gobernanza híbrida  

El objetivo no es la novedad.  
El objetivo es la fiabilidad estructural.
