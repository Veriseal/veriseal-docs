# Hypothèses de Sécurité

VeriSeal repose sur les hypothèses suivantes :

- SHA-256 reste résistant aux collisions
- Les schémas de signature restent sécurisés
- La sérialisation canonique est déterministe
- Les clés privées sont protégées sous gouvernance institutionnelle
- La politique de registre en mode ajout uniquement est appliquée opérationnellement

La sécurité est cryptographique, mais la résilience dépend de la gouvernance.
